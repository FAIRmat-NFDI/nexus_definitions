=================
NXapm_hit_finding
=================

**Status**:

  base class, extends :ref:`NXprocess`

**Description**:

  Base class for the configuration and results from a hit finding algorithm.

**Symbols**:

  The symbols used in the schema to specify e.g. dimensions of arrays.

  **n_dld**: Number of delay-line wires of the detector.

  **n_ht**: Number of hit qualities (hit types) distinguished.

  **p**: Number of pulses collected in between start_time and end_time resolved by an instance of :ref:`NXevent_data_apm`. If this is not defined, p is the number of ions included in the reconstructed volume if an application definition is used to store results of already reconstructed datasets.
